← Home
01

Core Concept

Exam One-liner

AWS WAF operates at Layer 7 (HTTP/HTTPS) to inspect and filter web traffic using rules defined in a Web ACL. It protects against SQL injection, XSS, bad bots, and abusive IPs — and can be deployed at edge (CloudFront) or regionally (ALB, API Gateway).

⚠

WAF is NOT for DDoS protection. For DDoS, use AWS Shield Standard (free, automatic) or AWS Shield Advanced (paid, with 24/7 DRT support). WAF + Shield Advanced together = complete L3/L4/L7 protection.

02

Where WAF Can Be Deployed

Application Load Balancer

Regional. Localized rules per ALB. Protects EC2, ECS, on-prem apps behind ALB.

API Gateway

Regional or edge-optimized. Protects REST and HTTP APIs.

CloudFront

Global edge (all PoPs). Protects CLB, EC2, S3 websites, custom origins. Lowest latency.

AWS AppSync

Protects GraphQL APIs from malicious queries.

Cognito User Pools

Protects authentication endpoints from brute force and credential stuffing.

⚠

Scope matters for the exam: A Web ACL created for CloudFront must be in us-east-1 (global scope). Web ACLs for ALB, API Gateway, AppSync, and Cognito are regional and must be in the same region as the resource.

03

Web ACL & Rules

Exam Key

A Web ACL is the container for WAF rules. Rules are evaluated in priority order (lowest number first). The first matching rule's action applies. If no rule matches, the default action (Allow or Block) applies.

What rules can match on

  • IP addresses — source IP or IP sets (up to 10,000 IPs per set)
  • HTTP headers — User-Agent, Referer, custom headers
  • HTTP body — first 8 KB inspected (configurable up to 64 KB)
  • URI strings / query strings
  • SQL injection — detect SQLi patterns
  • XSS — detect cross-site scripting patterns
  • Geo match — allow/block by country
  • Size constraints — e.g. body > 8 KB → block
  • Rate-based — count requests per 5-min window

Rule types

  • Regular rules — match conditions → single action
  • Rate-based rules — count requests from an IP; trigger when threshold exceeded in 5-min window
  • Rule Groups — reusable set of rules (your own or managed)
  • Managed Rule Groups — pre-built by AWS or Marketplace sellers
  • Rules have a WCU (WAF Capacity Unit) cost — max 5,000 WCUs per Web ACL

Rule Actions

ALLOW

Forward request to protected resource

BLOCK

Return HTTP 403 Forbidden to client

COUNT

Count match, continue evaluating rules. Used for testing.

CAPTCHA

Present CAPTCHA puzzle to verify human

CHALLENGE

Silent JS challenge to detect bots (no user interaction)

💡

COUNT is the testing action: Use COUNT mode first when deploying new rules — it logs matches without blocking traffic. Switch to BLOCK only after validating there are no false positives.

Rate-based rules — key details

  • Count requests from a single IP over a 5-minute rolling window
  • Minimum threshold: 100 requests
  • When threshold exceeded → action triggers (Block, CAPTCHA, etc.)
  • Rate-based rules can also scope by header, query string, URI in addition to IP
  • Automatically resets when request rate drops below threshold
  • Use case: brute force protection, scraping prevention, API abuse mitigation

Managed Rule Groups

Exam Key

Over 190 managed rules maintained by AWS and AWS Marketplace sellers. You subscribe to a rule group and add it to your Web ACL — AWS handles updates. You can set individual rules to Count or Override without modifying the group.

Baseline Rule Groups

💡

Overriding rules: You can set individual rules within a managed group to COUNT instead of their default action — useful for disabling a rule causing false positives without removing the entire group.

05

Logging

CloudWatch Logs

Up to 5 MB/sec. Near real-time. Log group name must start with aws-waf-logs-.

Amazon S3

5-minute intervals. Cheapest for long-term retention. Bucket name must start with aws-waf-logs-.

Kinesis Data Firehose

Near real-time. Can fan out to S3, Redshift, OpenSearch. Limited by Firehose quotas. Best for analytics pipelines.

Log filtering (Redacted Fields)

  • You can redact sensitive fields (e.g. passwords in HTTP body) from WAF logs
  • Configure log filters to only log matching or non-matching requests (reduce volume)
  • Logs include: timestamp, rule matches, action taken, request headers, IP, URI
  • Enable logs at the Web ACL level — applies to all associated resources

Architecture Pattern — CloudFront + ALB Origin Security ★

High-Exam-Frequency Pattern

Prevent users from bypassing CloudFront (and its WAF) by directly hitting the ALB. Solution: use a secret custom HTTP header set by CloudFront, validated by a WAF rule on the ALB. Rotate the header value with Secrets Manager + Lambda.

Users
→
WAF (CloudFront)
→
CloudFront adds X-Origin-Verify: <secret>
→
WAF (ALB) — checks header
→
ALB → EC2
Direct ALB access (no header)
→
WAF BLOCKS — 403
Secret rotation:
Secrets Manager auto-rotate
→
Lambda invoked
→
Updates CloudFront header + WAF rule

Why this pattern exists

  • CloudFront DNS is public — attackers can find the ALB endpoint separately
  • Direct ALB access bypasses WAF, geo-restrictions, and caching
  • ALB security group cannot restrict by arbitrary IPs (CloudFront IPs change)
  • Custom header + WAF rule is the standard AWS-recommended solution

Implementation steps

  • Store secret in Secrets Manager
  • CloudFront adds X-Origin-Verify: <secret> to all origin requests
  • ALB WAF rule: BLOCK if header absent or wrong value
  • Secrets Manager rotation → Lambda updates both CloudFront and WAF rule atomically
  • ALB security group: only allow CloudFront managed prefix list as backup layer
07

AWS Firewall Manager — Org-Wide WAF

Exam Key

Firewall Manager centrally deploys and manages WAF Web ACLs (and other firewall policies) across all accounts in an AWS Organization. New accounts get WAF policies automatically when they join.

What Firewall Manager manages

  • WAF Web ACL policies — push common rules org-wide
  • AWS Shield Advanced — enable and manage protection
  • VPC Security Groups — enforce baseline SG policies
  • Network Firewall — centralize VPC-level traffic inspection
  • Route 53 Resolver DNS Firewall — DNS-level protection

Requirements

  • AWS Organizations must be enabled with All Features
  • Designate a Firewall Manager administrator account
  • AWS Config must be enabled in all accounts (Firewall Manager uses it)
  • Firewall Manager policies enforce: existing + new accounts automatically
💡

Exam pattern: "Deploy WAF rules across 50 accounts without touching each one" → AWS Firewall Manager. It's the org-wide WAF orchestration layer, analogous to how Organizations + Config is to compliance.

08

WAF vs Shield vs Network Firewall

Frequently tested together — know exactly which layer and threat each service addresses.

ServiceLayerProtects AgainstKey Detail
AWS WAF Layer 7 (HTTP) SQLi, XSS, bad bots, rate abuse, geo Rules on content. Deploy on ALB, API GW, CloudFront, AppSync, Cognito
Shield Standard Layer 3 / 4 SYN floods, UDP reflection, volumetric DDoS Free, automatic, no config needed. Everyone gets it.
Shield Advanced L3 / L4 / L7 Sophisticated DDoS + L7 with WAF $3,000/month. DRT 24/7, cost protection, attack forensics, auto WAF rules
Network Firewall Layer 3 / 4 / 7 VPC ingress/egress, IPS/IDS, domain filtering Stateful firewall for VPC. Inspect traffic between VPCs, internet, on-prem
Security Groups / NACLs Layer 3 / 4 IP/port filtering SGs: stateful, instance level. NACLs: stateless, subnet level.

Exam Cheat Sheet

⚠

WAF ≠ DDoS protection. Use Shield Standard (free) or Shield Advanced for DDoS.

⚠

CloudFront WAF Web ACL must be in us-east-1. All others are regional.

✓

COUNT action = safe testing mode. Never blocks. Always use before BLOCK on new rules.

✓

Rate-based rules: 5-min window, minimum 100 req threshold, auto-resets.

✓

CloudFront + WAF + ALB + Secrets Manager = standard origin protection pattern. Memorize it.

✓

Firewall Manager = centralized WAF + Shield across all org accounts. Requires Config enabled.

✓

Bot Control Managed Rule Group: Common level (basic) vs Targeted level (ML-based, higher cost).

✓

Rules evaluated in priority order. First match wins. Default action if no match.

✓

Override managed rule action to COUNT to suppress false positives without removing the group.

✓

WAF logs: CloudWatch (5 MB/s), S3 (5-min batch), Firehose → S3/Redshift/OpenSearch.
