01

Core Concept

Exam One-liner

AWS Firewall Manager is a centralized security policy management service that automatically applies firewall rules (WAF, Shield, SGs, Network Firewall, DNS Firewall) across all accounts in an AWS Organization — including accounts that join in the future.

Core value proposition

  • Define a security policy once → auto-applied to all current and future accounts
  • Policies created at the region level (except Shield Advanced — global)
  • New resources created in any account get rules applied automatically
  • Centralized compliance dashboard — see which accounts are non-compliant
  • Each policy type covers different AWS services and resource types

Prerequisites (all required)

  • AWS Organizations with All Features enabled
  • Designate a Firewall Manager administrator account (not necessarily management acct)
  • AWS Config enabled in every member account and region (Firewall Manager uses it for compliance checking)
  • For Shield Advanced: subscribe to Shield Advanced in the admin account
⚠

Config is mandatory. If AWS Config is not enabled in an account/region, Firewall Manager cannot assess or enforce compliance there. This is a common exam distractor — Config must be pre-enabled.

02

Security Policy Types ★ Most Tested

Policy Type AWS WAF
  • Enforce WAF Web ACLs on ALBs, API Gateways, CloudFront across all accounts
  • Choose: apply a common Web ACL, or allow accounts to add their own rules within a managed baseline
Identify Only

Create the WebACL in each account — do NOT associate it to any resources. Assessment mode.

Auto Remediate

Create WebACL AND automatically associate it to all matching existing resources.

New Resources

Automatically apply WebACL to any new ALB/API GW/CloudFront resource as it is created.

Policy Type AWS Shield Advanced
  • Enforce Shield Advanced protection across all accounts on: ALB, CLB, NLB, Elastic IP, CloudFront
  • Two modes: view compliance only (assess impact) or auto remediate
  • Includes access to Shield Response Team (SRT) and cost protection for DDoS-related scaling
  • Auto-creates WAF Web ACLs to respond to active DDoS events (Shield Advanced + WAF integration)
Policy Type Security Groups — 3 Sub-types

Common Security Groups

  • Apply a common baseline SG to all EC2, ALB, and ENI resources org-wide
  • Enforces that a specific SG is always attached

Auditing Policy

  • Check that SG rules comply with your defined standards across all accounts
  • Detects overly permissive rules (e.g. 0.0.0.0/0 on port 22)

Usage Audit Policy

  • Identify unused or redundant SGs across the org
  • Optionally auto-clean up orphaned SGs
Policy Type AWS Network Firewall
  • Centrally manage VPC-level stateful/stateless firewalls across all accounts in the org
  • Provides L3–L7 inspection: IP/port filtering, protocol detection, domain-based filtering, IPS/IDS
Distributed

Creates and maintains a firewall endpoint in each VPC. Traffic stays local.

Centralized

Single firewall endpoint in a central/inspection VPC. Traffic routed through it via TGW.

Import Existing

Onboard existing Network Firewalls using Resource Sets. Manage them via Firewall Manager going forward.

Policy Type Route 53 Resolver DNS Firewall
  • Manage associations between DNS Firewall Rule Groups and VPCs across all accounts
  • Block DNS queries to known malicious domains; allow only approved domains (allowlist)
  • Prevents DNS exfiltration — data smuggled via DNS queries to attacker-controlled domains
  • Rule groups defined centrally, applied to every VPC in the org automatically
03

WAF vs Firewall Manager vs Shield — Decision Guide

These three services are always tested together. Memorize the decision logic.

ScenarioCorrect ServiceWhy
Protect a single ALB from SQLi/XSS WAF alone Single resource, granular rule control. No org management needed.
Apply WAF rules to all ALBs across 50 accounts Firewall Manager + WAF Multi-account enforcement. Firewall Manager deploys WAF policy org-wide.
Frequent DDoS attacks on your infrastructure Shield Advanced DRT access, attack forensics, cost protection, automatic DDoS mitigation rules.
Enforce Shield Advanced across all accounts in org Firewall Manager + Shield Advanced Firewall Manager enforces Shield Advanced policy across every account automatically.
Basic DDoS protection at no cost Shield Standard Free, automatic for all AWS customers. No configuration needed.
Enforce consistent SG rules across org Firewall Manager (SG policy) Common SG or Auditing policy type enforces baseline rules org-wide.
Inspect VPC-level traffic (L3–L7) org-wide Firewall Manager + Network Firewall Network Firewall policy type in Firewall Manager — distributed or centralized mode.
💡

Exam rule: If the scenario says "single account / single resource" → WAF (or Shield). If it says "all accounts", "organization", "new accounts automatically", or "centrally manage" → Firewall Manager is always part of the answer.

04

Compliance Enforcement & Remediation

Exam Key

Every Firewall Manager policy has two dimensions: scope (which accounts/resources it applies to) and remediation mode (identify only vs auto-remediate). New resources created in any scoped account are automatically brought into compliance.

Policy scope options

  • All accounts in the Organization
  • Specific OUs — apply to Dev OU but not Prod OU
  • Exclude accounts — all except listed accounts
  • Filter by resource tags — e.g. only resources tagged Env: Prod
  • Filter by resource type — e.g. only ALBs, not API Gateways

Compliance dashboard

  • Shows compliant/non-compliant status per account per policy
  • Drill into which specific resources are out of compliance
  • Integrates with AWS Security Hub for centralized findings
  • Violations reported via AWS Config findings
  • Non-compliance does NOT mean the resource is blocked — only flagged (unless auto-remediate is on)
05

Shield Advanced — Key Details

What Shield Advanced adds over Standard

  • Shield Response Team (SRT) — 24/7 dedicated DDoS experts
  • Advanced reporting — attack diagnostics, forensics, near real-time metrics
  • Cost protection — credits for scaling costs incurred during a DDoS
  • Automatic application layer (L7) mitigation — auto-creates WAF rules during an attack
  • Proactive engagement — SRT can contact you during active attacks
  • Global Threat Dashboard — visibility into current attack landscape

Pricing & scope

  • $3,000 / month per organization (not per account)
  • Covers: ALB, CLB, NLB, Elastic IP, CloudFront, Route 53, Global Accelerator
  • Must explicitly enable protection per resource or use Firewall Manager to automate
  • 1-year commitment required
  • WAF is included at no extra WAF cost for Shield Advanced-protected resources
⚠

Shield Standard vs Advanced: Shield Standard is automatic and free for all accounts — it protects against L3/L4 volumetric attacks. Shield Advanced requires opt-in and payment — it adds L7 protection, SRT, cost protection, and reporting.

Exam Cheat Sheet

⚠

Config must be enabled in every account/region before Firewall Manager can work there.

⚠

"Single resource" → WAF/Shield directly. "Org-wide / all accounts" → Firewall Manager.

✓

Policies created at region level. Exception: Shield Advanced policy is global in scope.

✓

New accounts joining org + new resources created → automatically covered by Firewall Manager policies.

✓

WAF policy has 3 remediation modes: Identify only / Auto remediate existing / Auto apply to new.

✓

SG policy has 3 sub-types: Common (enforce baseline) / Audit (check rules) / Usage Audit (find orphans).

✓

Network Firewall modes: Distributed (per VPC) vs Centralized (inspection VPC via TGW).

✓

Shield Advanced: $3K/month/org, SRT access, cost protection, auto L7 WAF rules during attacks.

✓

DNS Firewall via Firewall Manager: blocks DNS to malicious domains org-wide, prevents DNS exfiltration.

✓

Firewall Manager admin account ≠ management account. Can be a dedicated security member account.
