01

Core Concept

Exam One-liner

Trusted Advisor provides account-level best practice recommendations across 6 categories. The number of checks available depends entirely on your AWS Support plan. Automation is done via EventBridge (check status changes) and the Support API (Business/Enterprise only).

What it assesses

  • Checks your account against AWS best practices — not real-time threats
  • Produces checks with status: OK, Warning, Error, or Not Available
  • Refreshes checks weekly automatically (Business/Enterprise: can trigger via API)
  • Covers 500+ checks total (most require Business/Enterprise Support)
  • Operates at the account level — not across Organizations by default

What it does NOT do

  • Does not detect threats — use GuardDuty for that
  • Does not scan for CVEs — use Inspector for that
  • Does not enforce compliance — use Config for that
  • Does not auto-remediate — you act on recommendations manually or via automation
  • Does not work natively across multiple accounts (use Support API + aggregation)

Support Plan Access ★ Most Tested

⚠

Number-one exam trap: The access level for Trusted Advisor checks is entirely gated by Support plan. Basic/Developer get only 7 core checks. Business/Enterprise get all checks + Support API access.

Support Plan Checks Available Support API Programmatic Refresh EventBridge
Basic / Developer 7 core checks only (Security + Service Limits subset) ✗ ✗ Limited
Business ✓ All checks (500+) ✓ ✓ via API ✓ Full
Enterprise On-Ramp ✓ All checks (500+) ✓ ✓ via API ✓ Full
Enterprise ✓ All checks (500+) ✓ ✓ via API ✓ Full

7 Core checks (Basic/Developer — free)

  • Security — S3 bucket permissions (public?)
  • Security — Security Groups — specific ports unrestricted
  • Security — IAM use (is IAM being used at all?)
  • Security — MFA on Root Account
  • Security — EBS public snapshots
  • Security — RDS public snapshots
  • Service Limits — Service limits (1 check)
03

Six Check Categories

Cost Optimization
Reduce spend
  • Idle EC2 instances (low CPU/network utilization)
  • Unassociated Elastic IP addresses
  • Underutilized EBS volumes
  • Reserved Instance optimization recommendations
  • Unattached load balancers
Performance
Improve responsiveness
  • High-utilization EC2 instances
  • CloudFront content delivery optimization
  • EBS throughput and latency checks
  • RDS instance type recommendations
Security
Close security gaps
  • IAM use, MFA on root, password policy
  • S3 bucket public access
  • Security group open ports (0.0.0.0/0)
  • CloudTrail logging enabled
  • Exposed access keys (GitHub, public repos)
Fault Tolerance
Increase resilience
  • EBS snapshots (last snapshot age)
  • EC2 instances across AZs
  • RDS Multi-AZ enabled
  • Auto Scaling group health checks
  • Route 53 failover records
Service Limits
Prevent throttling
  • 50+ service quotas monitored
  • Alerts at 80% and 100% of limit reached
  • Also shows "unable to retrieve data" status
  • Triggers EventBridge for quota automation
Operational Excellence
Improve operations
  • CloudFormation stack status checks
  • IAM Access Analyzer findings
  • AWS well-architected framework alignment
  • CodeBuild project configurations

For Service Limits specifically, the states are: Unable to retrieve data (grey) → 80% reached (yellow) → 100% reached (red). These map to EventBridge event thresholds.

05

EventBridge Integration (Monitoring Architecture)

Exam Key

Trusted Advisor publishes events to EventBridge when check status changes. This is the standard automation pattern — react to check status changes without polling the Support API.

Monitoring architecture — Low Utilization EC2
Trusted Advisor → Check status change event → EventBridge Rule → SNS → notify ops team
Trusted Advisor → Check status change event → EventBridge Rule → Lambda → auto-remediate
Service Quota monitoring architecture
ServiceLimitUsage metric → Trusted Advisor (80% / 100%) → EventBridge → Lambda → request quota increase

EventBridge event payload fields

  • source: "aws.trustedadvisor"
  • detail-type: "Trusted Advisor Check Item Refresh Notification"
  • status: WARN / ERROR / OK / NOT_AVAILABLE
  • check-name: e.g. "Low Utilization Amazon EC2 Instances"
  • resource_id: ARN of the affected resource
  • region: region of the resource

Service Quotas event fields

  • MetricName: ServiceLimitUsage
  • Region: affected region (e.g. us-east-1)
  • ServiceName: e.g. EC2
  • ServiceLimit: e.g. "EC2 On-Demand Instances"
  • Triggers at: 80% (yellow), 100% (red), or unable to retrieve (grey)
06

Support API (Business / Enterprise Only)

Exam Key

The AWS Support API gives programmatic access to Trusted Advisor checks — useful for multi-account dashboards, automated refresh, and exporting check results to external tools.

Key Support API operations

  • DescribeTrustedAdvisorChecks — list all available checks
  • DescribeTrustedAdvisorCheckResult — get current status and resources for a check
  • RefreshTrustedAdvisorCheck — trigger an immediate refresh of a specific check
  • DescribeTrustedAdvisorCheckSummaries — aggregated status across checks

Multi-account aggregation pattern

  • Trusted Advisor is per-account only — no native org aggregation
  • Pattern: Lambda calls DescribeTrustedAdvisorCheckResult for each account via cross-account roles
  • Aggregate results to S3 → Athena or custom dashboard
  • Schedule via EventBridge Scheduled Rules (cron)
⚠

API region requirement: The Support API endpoint is us-east-1 only (support.us-east-1.amazonaws.com). Calls from any region must target us-east-1 — this is a common exam gotcha for multi-region automation scripts.

Trusted Advisor vs Service Quotas Service

Both deal with service limits — the exam tests which to use when.

Alerts EventBridge at 80% and 100% CloudWatch alarms on quota usage metrics

Automation EventBridge → Lambda (react to breach) Lambda → RequestServiceQuotaIncrease API (proactive)
Required support plan Business/Enterprise for full access Available to all (free)
Best for exam "Alert when approaching limit" → Trusted Advisor + EventBridge "Automatically increase quota" → Service Quotas API
08

Exam Cheat Sheet

⚠

Basic/Developer = only 7 checks. "All checks" = Business or Enterprise Support required.

⚠

Support API = us-east-1 only. Multi-region scripts must explicitly target that endpoint.

✓

Checks refresh weekly automatically. Business/Enterprise can force refresh via RefreshTrustedAdvisorCheck API.

✓

Automation pattern: Trusted Advisor status change → EventBridge → SNS / Lambda. Not polling.

✓

Service Limits checks: EventBridge triggers at 80% (yellow) and 100% (red). MetricName = ServiceLimitUsage.

✓

Trusted Advisor is per-account. For org-wide reporting, aggregate via Lambda + Support API + cross-account roles.

✓

"Exposed IAM access key in GitHub" → Trusted Advisor Security check (not GuardDuty, not Inspector).

✓

Trusted Advisor does not remediate. Use EventBridge → Lambda to act on findings.
