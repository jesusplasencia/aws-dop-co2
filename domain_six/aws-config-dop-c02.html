← Home
01

Core Concept

Exam One-liner

AWS Config records the configuration state of AWS resources over time and evaluates them against compliance rules — it does not prevent changes.

What it does

  • Records resource configurations as Configuration Items
  • Tracks changes over time (timeline view)
  • Evaluates compliance against rules
  • Correlates with CloudTrail API calls
  • Aggregates across regions and accounts

Questions Config answers

  • Is there unrestricted SSH access to my security groups?
  • Do my S3 buckets have public access?
  • How has my ALB configuration changed over time?
  • Are all EC2 instances tagged correctly?
  • Are EBS volumes encrypted?
⚠

Key exam trap: AWS Config rules are detective, not preventive. They flag non-compliance after the fact. Use SCPs or IAM to actually prevent actions.

Outputs & Integrations

S3 (configuration history) Athena (query S3 data) SNS (all events / filtering) EventBridge (non-compliant triggers) SSM Automation (remediation) Lambda (custom rules)
02

Configuration Recorder

Exam Key

The Configuration Recorder is the engine that captures resource state. It must exist before Config can track anything. It is created automatically when you enable Config via Console or CLI.

Configuration Item (CI)

  • Point-in-time snapshot of a resource
  • Captures: attributes, relationships, config, events
  • Created on every detected change
  • Stored permanently in S3

Custom Recorder

  • Record only specific resource types to reduce cost
  • Useful for large-scale deployments
  • Reduces noise and configuration item volume
🔒

SCP protection pattern: Use an SCP to deny config:DeleteConfigurationRecorder, config:StopConfigurationRecorder, and config:DeleteDeliveryChannel — preventing member accounts from disabling Config.

03

Config Rules

AWS Managed Rules (75+)

  • Pre-built by AWS, ready to enable
  • Examples: restricted-ssh, s3-bucket-public-read-prohibited, iam-password-policy, access-keys-rotated
  • Configurable via input parameters

Custom Config Rules

  • Backed by a Lambda function
  • You define the evaluation logic
  • Examples: EBS must be gp2, EC2 must be t2.micro
  • Triggered by: config change, periodic schedule, or on-demand

Rule Evaluation Triggers

Configuration Change — triggered when resource changes Periodic — scheduled (e.g. every 24h) On-demand — manual evaluation
⚠

Remember: Rules evaluate compliance but do not block or rollback any action. Combine with Remediations to automate response.

04

Remediations

Exam Key

Remediations use SSM Automation Documents to automatically fix non-compliant resources. You can configure Remediation Retries if the resource remains non-compliant after initial auto-remediation.

Non-compliant resource → AWS Config detects → Triggers Remediation → SSM Automation Doc → Resource fixed ✓

Remediation Options

  • AWS-Managed Automation Documents — pre-built SSM docs (e.g. AWSConfigRemediation-RevokeUnusedIAMUserCredentials)
  • Custom Automation Documents — invoke a Lambda function for complex logic
  • Remediation Retries — set a retry count (e.g. 5) if still non-compliant after first attempt
  • Manual vs Auto remediation — auto runs immediately; manual requires approval
05

Notifications

EventBridge (preferred)

  • Triggers on non-compliant resource events
  • Route to Lambda, SNS, SQS, Step Functions, etc.
  • Granular event filtering
  • Best for automated response pipelines

SNS

  • Receives all events (config changes + compliance state)
  • Use SNS Filtering or filter at the client side
  • Useful for ops team alerting
06

Aggregators

Exam Key

An Aggregator collects Config data from multiple accounts and/or regions into one central account for a unified compliance view. With AWS Organizations, no individual account authorization is needed.

Important details

  • Created in a dedicated aggregator account (not necessarily management account)
  • Source accounts must authorize the aggregator — unless using AWS Organizations
  • Rules are still defined and evaluated per-account, not centrally enforced by the aggregator
  • Deploy rules to all accounts using CloudFormation StackSets
  • Provides aggregated view of compliant/non-compliant resources across the org
⚠

Exam trap: An Aggregator collects and displays data — it does not deploy rules or remediate resources. Rules must exist in each source account independently.

07

Conformance Packs

Exam Key

A Conformance Pack is a bundle of Config rules + remediation actions deployed as a single YAML template (similar to CloudFormation). Deploy to an account, a region, or across an entire AWS Organization.

Features

  • YAML-formatted template — includes rules and remediations together
  • Can include custom Config Rules backed by Lambda
  • Accept input parameters for flexibility (e.g. min password length, max key age)
  • Deploy via CodePipeline CI/CD for GitOps compliance
  • Designate a Delegated Administrator to deploy packs to an AWS Organization (can be a member account)
  • Pre-built sample packs available (PCI-DSS, CIS, HIPAA-aligned)

08

Organizational Rules vs Conformance Packs

This comparison is a high-probability exam question — know it cold.

Feature Organizational Rules Conformance Packs
Scope AWS Organization AWS Accounts & AWS Organization
Rules Count One rule per deployment Many rules at once
Evaluation Type Predefined rules, enforced at Org level Predefined rules, enforced at Account level
Compliance Level Managed at the Organizational level Managed at the Account level
Deployment Format Via AWS Organizations console/API YAML template (CloudFormation-style)
Custom Rules Yes (Lambda) Yes (Lambda)
Remediation Not included in definition Included in pack template

Config does NOT prevent non-compliant actions — use SCP/IAM for prevention

✓

Aggregators collect data, they don't deploy rules or fix resources

✓

Conformance Pack = many rules + remediations bundled in one YAML

✓

EventBridge for automated response; SNS for ops notifications

✓

Protect Config with SCP denying Delete/Stop recorder actions

✓

Custom rules need a Lambda function; can also be in Conformance Packs

✓

With AWS Organizations, no per-account authorization needed for aggregation

✗

Remediation uses SSM Automation, not Lambda directly (Lambda is used inside SSM docs)
