01

Core Concept

Exam One-liner

Amazon Detective is a post-detection forensic investigation tool. It does not detect threats — GuardDuty, Macie, and Security Hub do that. Detective answers "why did this happen and what was affected?" by building a persistent ML-based behavior graph from ingested event data.

What Detective does

  • Automatically collects and processes VPC Flow Logs, CloudTrail, GuardDuty findings
  • Builds a behavior graph — a time-series model of entity relationships and activity
  • Uses ML and statistical analysis to establish baselines and flag deviations
  • Produces interactive visualizations — who did what, from where, when
  • Retains up to 1 year of aggregated data for trend analysis
  • Surfaces related findings and context across IP, account, role, instance

What Detective does NOT do

  • Does not detect threats — it investigates findings already raised
  • Does not block or remediate — purely analytical / read-only
  • Does not replace CloudTrail, VPC Flow Logs — it ingests them independently
  • Does not generate alerts or send notifications
  • Does not query raw logs directly — it works on the processed behavior graph
⚠

Core exam trap: GuardDuty finds the threat. Detective investigates it. If a question asks "identify root cause," "investigate suspicious activity," or "understand scope of a compromise" → Amazon Detective, not GuardDuty.

02

How It Works

1
Security finding raised
GuardDuty, Macie, or Security Hub identifies a potential issue (e.g. compromised EC2, unusual IAM activity)
2
Detective continuously ingests event data
VPC Flow Logs, CloudTrail management/data events, GuardDuty findings — ingested automatically, no manual setup per source
3
Behavior graph built with ML
Entities (IPs, IAM users, EC2 instances, roles) are modeled with their normal activity patterns over time. Deviations are surfaced automatically.
4
Analyst pivots from finding to graph
From a GuardDuty finding, analyst opens Detective to see: which IPs communicated with the instance, what API calls were made, which roles were involved, what was unusual vs baseline
5
Root cause identified → action taken
Analyst determines blast radius, affected resources, and entry point — then initiates remediation (quarantine, credential revocation, etc.)
03

Data Sources

VPC Flow Logs

Network traffic patterns — connections, ports, bytes transferred. Maps IP-to-instance communications.

CloudTrail

API calls, console actions, IAM activity. Tracks who called what API, from where, at what time.

GuardDuty Findings

Enriches existing findings with behavioral context. Correlates findings to the same entity over time.

EKS Audit Logs

Kubernetes API activity — maps K8s user/role actions for cluster investigations.

💡

Detective ingests these sources automatically once enabled — you do not need to configure CloudTrail or VPC Flow Logs delivery separately. Data is retained for up to 1 year in the behavior graph, enabling long-term trend analysis.

04

Behavior Graph & Visualizations

Exam Key

The behavior graph is Detective's core data structure — a time-series model linking entities (AWS accounts, IAM users/roles, EC2 instances, IPs, S3 buckets) and their interactions. Visualizations let analysts navigate entity relationships without writing queries.

IP address panel

  • Which EC2 instances communicated with this IP
  • Volume of traffic over time vs baseline
  • Geo-location and threat intelligence context
  • First seen / last seen timestamps

IAM user / role panel

  • All API calls made by the entity
  • Normal vs anomalous API call frequency
  • New APIs called never used before
  • Assumed roles and cross-account activity

EC2 instance panel

  • All inbound/outbound network connections
  • Associated GuardDuty findings
  • Port activity over time
  • Processes (if SSM agent data available)

Key investigation questions Detective answers

Is this API call normal for this IAM user? Has this IP communicated before? What else did the compromised role access? When did anomalous behavior start? How many resources were affected? Was this a one-time event or ongoing pattern? What was the blast radius of this finding? Which account/region is the entry point?
05

Multi-Account & Organizations

Account model

  • One account is the Administrator (behavior graph owner)
  • Other accounts are member accounts — their data is ingested into the admin graph
  • Integrates with AWS Organizations — auto-enroll member accounts
  • Administrator sees a unified behavior graph across all member accounts
  • Member accounts cannot access the behavior graph — only the admin can

Integration with GuardDuty

  • From a GuardDuty finding, analysts can pivot directly into Detective with one click
  • Detective pre-populates the investigation with the relevant entities from the finding
  • Both services support the same delegated administrator account pattern
  • GuardDuty findings appear as enriched entities in Detective's graph — with historical context
06

Detective in the Security Stack ★ Most Tested

The exam tests whether you know which service handles which phase of a security incident lifecycle.

Detect

  • GuardDuty — runtime threat detection (ML, anomaly, threat intel)
  • AWS Macie — sensitive data exposure in S3
  • AWS Inspector — vulnerabilities (CVEs) in EC2/Lambda/containers
  • Security Hub — aggregated findings dashboard

Investigate

  • Amazon Detective — root cause analysis, blast radius, behavior graph
  • Answers "what happened, how, who was involved"
  • ML visualizations of entity relationships
  • Up to 1 year of behavioral history

Respond

  • EventBridge + Lambda — automated remediation
  • WAF / Security Groups / NACLs — block traffic
  • IAM — revoke credentials
  • AWS Systems Manager — patch, isolate instances
ServicePhaseCore capabilityKey distinction
GuardDuty Detect Threat detection via ML on CloudTrail, VPC Flow Logs, DNS Generates findings. Does not investigate or remediate.
Amazon Detective Investigate Root cause analysis via behavior graph and ML visualizations Answers "why" and "how far." No detection, no blocking.
Security Hub Aggregate Normalize and centralize findings from multiple sources SIEM-like view. Does not detect or investigate independently.
AWS Macie Detect Sensitive data (PII) classification in S3 Data classification only. Not threat behavior detection.
AWS Inspector Detect Vulnerability scanning (CVEs) in EC2, Lambda, containers Pre-exploit vulnerability assessment. Not runtime detection.
CloudTrail Audit Record all API calls for compliance and forensics Raw log storage. Detective and GuardDuty consume CloudTrail data.
