EOverview
What it is
TypeServerless event bus — routes events between AWS services, SaaS apps, and custom applications
Event formatJSON — every event has a standard envelope: source, detail-type, detail, account, region, time
ThroughputDefault 10,000 events/sec per region (soft limit, can be raised)
LatencyTypically under 500ms end-to-end
Old nameCloudWatch Events (same underlying service, rebranded)

  "detail-type": "EC2 Instance State-change Notification",

RRules & Triggers
Trigger Type 1 — Event Pattern
  • Rule matches incoming events based on their JSON content
  • Pattern is a partial JSON subset of the event — only specify fields you care about
  • Supports content filtering (prefix, suffix, numeric ranges, etc.)
  • Multiple targets can be invoked by a single rule
// Match any EC2 instance termination in us-east-1
{
  "source": ["aws.ec2"],
  "detail-type": ["EC2 Instance State-change Notification"],
  "detail": {
    "state": ["terminated"]
  }
}
⚡ EXAM TIP Classic pattern: IAM Root User sign-in → EventBridge rule → SNS email alert. Know this cold — it appears repeatedly.
Trigger Type 2 — Schedule
  • Rate expression: rate(5 minutes), rate(1 hour)
  • Cron expression: full cron syntax for complex schedules
  • EventBridge Scheduler (newer): dedicated service for large-scale scheduling, one-time or recurring
  • Scheduler supports flexible time windows — spread invocations to avoid thundering herd
# Every day at 6 AM UTC
cron(0 6 * * ? *)

# Every 15 minutes
rate(15 minutes)
⚡ EXAM TIP EventBridge Scheduler can invoke any AWS API action directly as a target — not just Lambda. Replaces cron-based Lambda triggers for scheduled tasks.

BEvent Buses
Default Event Bus
  • Receives all AWS service events automatically
  • One per account per region
  • Cannot be deleted
  • AWS services publish here natively
Partner Event Bus
  • Receives events from SaaS partners
  • Partners: Zendesk, Datadog, PagerDuty, Auth0, Stripe…
  • Partner must be activated in AWS console
  • Creates a dedicated bus per partner
Custom Event Bus
  • For your own application events
  • Publish via PutEvents API
  • Scoped to an account by default
  • Share cross-account via resource-based policy
  • Can be shared with other AWS accounts or org
Cross-Account / Cross-Region Event Routing
Account A
Source Bus → EventBridge Rule
target = bus in Acct B → Account B
Central Event Bus → Lambda / SQS / etc.
  • Receiving bus must have a resource-based policy granting events:PutEvents from the source account
  • Cross-region: rule targets a bus in a different region — same account or cross-account
  • Common pattern: centralize all org events to a security / audit account
⚡ EXAM TIP Multi-account event aggregation = EventBridge rules in each account routing to a central custom bus in a dedicated audit account. This is the DevOps-recommended observability pattern.
Event Archive & Replay
  • Archive all events or a filtered subset from any bus
  • Configurable retention: indefinite or specific number of days
  • Replay: re-process archived events through a bus to recover from bugs or reprocess data
  • Replay can target a different bus or the same bus with a specific time window
⚡ EXAM TIP Archive + Replay is the EventBridge answer to "how do you reprocess events after fixing a Lambda bug?" — events are replayed from the archive, not resent by the source.
SSchema Registry
  • Automatically discovers and stores event schemas from your buses
  • Schema = the structure/shape of an event's detail field
  • Allows you to generate code bindings (Java, Python, TypeScript) for event types — speeds up development
  • Versioned — tracks schema changes over time
  • Available for AWS service events, partner events, and custom events
⚡ EXAM TIP Schema Registry is a secondary topic — know it exists and that it generates typed code bindings. Unlikely to be a primary question focus on DOP-C02.
FContent Filtering

Event patterns support advanced filtering so rules only trigger on precisely matched events — reducing unnecessary target invocations.

Prefix
String starts with value
{ "prefix": "us-east" }
Suffix
String ends with value
{ "suffix": ".jpg" }
Anything-But
Match anything except listed values
{ "anything-but": ["WARN","INFO"] }
Exists
Field is present (true) or absent (false)
{ "exists": true }
Numeric
Range comparison on numeric fields
{ "numeric": [">", 100, "<=", 500] }
Multiple / AND / OR
Combine filters; same field = OR, different fields = AND
{ "state": ["running","stopped"] }
⚡ EXAM TIP Content filtering happens before the target is invoked — you are not charged for filtered-out events that don't reach a target. Use anything-but to suppress noisy events like health checks.
↻Reliability & Error Handling
Retry Policy
  • EventBridge retries failed target invocations with exponential backoff
  • Default: 24 hours max retry window, up to 185 retries
  • Configurable per rule target: max event age + max retry attempts
  • Applies to async targets (Lambda async, SQS, etc.)
Dead Letter Queue (DLQ)
  • Configure an SQS queue as DLQ on a rule target
  • Events that fail all retries are sent to the DLQ
  • Prevents event loss — inspect and replay from DLQ
  • Each target has its own DLQ configuration
⚡ EXAM TIP EventBridge → Lambda failures → DLQ on EventBridge target (not on Lambda itself). Don't confuse with Lambda's own DLQ/SQS destination.
API Destinations
  • Invoke any external HTTP/S endpoint as an EventBridge target
  • Supports OAuth, API Key, and Basic auth via Connections (credentials stored in Secrets Manager)
  • Rate limiting: configure max invocations per second to protect downstream APIs
  • Use case: send AWS events to Slack, PagerDuty, Splunk, or any webhook
🔒Security & Permissions
IAM Role (for targets)
  • EventBridge needs an execution role to invoke targets on your behalf
  • Role must allow the target action (e.g. lambda:InvokeFunction, sqs:SendMessage)
  • AWS-managed targets (Lambda, SNS) also accept resource-based policies instead
Resource-Based Policy (on the Bus)
  • Controls who can put events onto a custom bus
  • Grant events:PutEvents to specific accounts, OUs, or entire AWS Organization
  • Required for cross-account event routing
{
  "Effect": "Allow",
  "Action": "events:PutEvents",
  "Principal": { "AWS": "arn:aws:iam::111222333444:root" },
  "Resource": "arn:aws:events:us-east-1:123456789012:event-bus/central"
}
