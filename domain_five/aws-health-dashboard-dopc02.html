VTwo Views
Service Health (Public)
  • Shows health of all AWS services across all regions
  • General / public status — not account-specific
  • Historical log of past 12 months of service interruptions
  • Available at health.aws.amazon.com/health/status
  • No login required — publicly accessible
  • Equivalent to the old "AWS Service Health Dashboard"
Your Account (Personal)
  • Formerly Personal Health Dashboard (PHD)
  • Personalized view — shows only events affecting your specific resources
  • Provides alerts + remediation guidance for impacting events
  • Shows scheduled maintenance, lifecycle notifications, security advisories
  • Proactive notifications for planned AWS changes
  • Can aggregate across an entire AWS Organization
⚡ EXAM TIP — KEY DISTINCTION Service Health = "Is EC2 down globally?" (public, generic). Your Account = "Are MY EC2 instances affected?" (private, resource-specific). The exam tests this distinction — always prefer Your Account / PHD when the scenario involves your own resources.
TEvent Types
Event TypeDescriptionExample
Issue Active service disruption or degraded performance affecting your resources EC2 API errors in us-east-1 affecting your instances
Scheduled Change Planned maintenance or lifecycle events requiring action or awareness EC2 instance retirement, RDS maintenance window, TLS cert deprecation
Account Notification Informational — billing, security advisories, service limit warnings IAM access key exposed in public repo, service limit approaching
Event Scope

ACCOUNT EVENTS

  • Specific to your account and resources
  • Only visible to your account
  • Examples: your EC2 scheduled for retirement, your IAM key exposed

PUBLIC EVENTS

  • Broad service-level events affecting many customers
  • Visible to all accounts
  • Examples: regional EC2 degradation, S3 connectivity issues
⚡ EXAM TIP EventBridge can react to both Account Events and Public Events from AWS Health. The source in EventBridge patterns is "aws.health".
⚡EventBridge Automation Patterns
How It Works
  • AWS Health publishes events to EventBridge on the default event bus
  • Source: "aws.health" — match on detail-type and detail.service
  • Works for both Account Events and Public Events
  • Enables fully automated remediation without human intervention
// EventBridge pattern — react to any AWS Health event
{
  "source": ["aws.health"],
  "detail-type": ["AWS Health Event"],
  "detail": {
    "service": ["EC2"],
    "eventTypeCategory": ["scheduledChange"]
  }
}
Key Automation Scenarios (exam-tested)
Exposed IAM Access Key AWS Health → EventBridge → Lambda → IAM: Delete Key
EC2 Scheduled for Retirement AWS Health → EventBridge → EC2: Restart Instance
Service Degradation Alert AWS Health → EventBridge → SNS → Email / Slack / PagerDuty
Capture Event for Audit AWS Health → EventBridge → Kinesis Firehose → S3 / OpenSearch
RDS Maintenance Window AWS Health → EventBridge → Lambda → Switch Traffic to Replica
⚡ EXAM TIP The two most-tested scenarios: (1) IAM Key Exposed → Lambda → Delete Key, and (2) EC2 Retirement → EventBridge → Restart/Replace Instance. Know these flows cold.
OOrganization-Level Health
  • Enable AWS Health Organizational View from the management (payer) account
  • Aggregates health events across all accounts in the AWS Organization
  • Requires AWS Organizations and Business/Enterprise Support on the management account
  • View events per-account, per-service, per-region across the org
  • EventBridge in the management account can react to health events from member accounts
  • Useful for: centralized ops team responding to issues across all child accounts
⚡ EXAM TIP "Monitor health of all accounts in an organization from a single place" → AWS Health Organizational View from the management account. Requires Business/Enterprise Support.

⚠ REQUIREMENT AWS Health API access requires Business or Enterprise Support plan. Calls from accounts with Basic support return an error. This is a common exam distractor.
