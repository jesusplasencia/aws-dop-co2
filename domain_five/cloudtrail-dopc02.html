COverview
What it is
PurposeGovernance, compliance, and operational auditing — records who did what and when in your AWS account
ScopeEvery API call made via Console, CLI, SDK, or AWS service-to-service
Enabled by defaultYes — Event History is always on (90-day rolling window, free)
TrailCustom configuration to persist events to S3 and/or CloudWatch Logs beyond 90 days
LatencyEvents typically appear within 15 minutes of the API call
Destinations
  • S3 bucket — long-term storage, query with Athena
  • CloudWatch Logs — real-time metric filters and alarms
  • EventBridge — react to any API call in real time
  • CloudTrail Lake — managed SQL query engine for CloudTrail events (no separate Athena setup needed)
⚡ EXAM TIP For long-term audit beyond 90 days → Trail → S3 + query with Athena. For real-time alerting on API calls → CloudTrail → EventBridge.
ECloudTrail Event Types
Management Events
CONTROL PLANE OPERATIONS
  • Operations on resources (create, modify, delete)
  • Logged by default — always on
  • Examples: CreateTrail, AttachRolePolicy, RunInstances, CreateBucket
  • Can split Read (ListBuckets, DescribeInstances) from Write (TerminateInstances, DeleteBucket)
  • No additional cost
Data Events
DATA PLANE OPERATIONS
  • Operations on data within resources — very high volume
  • NOT logged by default — must be explicitly enabled
  • Additional cost per event
  • S3: GetObject, PutObject, DeleteObject
  • Lambda: Invoke API
  • DynamoDB: GetItem, PutItem
  • Can filter by specific resource ARN
Insights Events
ANOMALY DETECTION
  • Detect unusual write API activity
  • Analyzes normal management events → builds a baseline
  • Continuously monitors for deviations
  • Detects: IAM action bursts, hitting service limits, inaccurate provisioning
  • Outputs to: CloudTrail Console, S3, EventBridge
  • Must be explicitly enabled — additional cost

TTrails
Trail Types
ScopeBehaviour
Single RegionLogs events in one specified region only
All RegionsLogs events across all current and future regions — recommended default
Org TrailCreated in management account — captures all events across all member accounts
⚡ EXAM TIP Always recommend all-region trail for compliance. Single-region trails miss global services like IAM and STS that log to us-east-1.
Trail Configuration Options
  • S3 bucket — required destination; optionally with SSE-KMS encryption
  • CloudWatch Logs — optional; enables metric filters + real-time alerting
  • SNS notifications — notify when new log file is delivered to S3
  • Log file validation — creates a digest file to verify logs haven't been tampered with
  • Event selectors — choose Management, Data, or Insights events per trail
  • Advanced event selectors — filter by resource ARN, resource type, event name
Event Retention

EVENT HISTORY (DEFAULT)

  • 90 days — free, always available in console
  • Management events only
  • No trail required

LONG-TERM (TRAIL)

  • Indefinite — persisted to S3
  • Set S3 lifecycle policies for cost management
  • Query with Athena (SQL) or CloudTrail Lake

⚡Intercepting API Calls with EventBridge

Every CloudTrail event is automatically available in EventBridge via the default event bus — no extra config needed. This enables real-time automated responses to any AWS API call.

Alert on critical resource deletion
User → DynamoDB DeleteTable → CloudTrail → EventBridge → SNS alert
Detect SG rule change
User → EC2 AuthorizeSecurityGroupIngress → CloudTrail → EventBridge → SNS / Lambda remediate
Auto-remediate IAM change
User → IAM CreateUser → CloudTrail → EventBridge → Lambda → DeleteUser
Archive for compliance
All API calls → CloudTrail Trail → S3 + SSE-KMS → Athena query
⚡ EXAM TIP CloudTrail + EventBridge is the answer to any question about reacting to a specific AWS API call in real-time. The EventBridge pattern uses "source": ["aws.cloudtrail"] or directly the service source (e.g. "aws.ec2") with specific eventName in the detail filter.
EventBridge Pattern for CloudTrail API Call
// Trigger on any IAM CreateUser call
{
  "source": ["aws.iam"],
  "detail-type": ["AWS API Call via CloudTrail"],
  "detail": {
    "eventSource": ["iam.amazonaws.com"],
    "eventName": ["CreateUser"]
  }
}
🔒Log Integrity & Security
Log File Validation
  • Enable on the trail to detect tampering or deletion of log files
  • CloudTrail creates a digest file every hour containing SHA-256 hashes of all delivered log files
  • Digest file is signed with an AWS private key — verifiable with public key
  • Validate with: aws cloudtrail validate-logs CLI command
  • Required for compliance frameworks (PCI-DSS, SOC, ISO 27001)
⚡ EXAM TIP "Prove CloudTrail logs haven't been modified" → Log File Validation (digest files). This is the only mechanism to cryptographically verify log integrity.
Encryption & Access Control
  • Logs in S3 encrypted with SSE-S3 by default
  • Use SSE-KMS for customer-managed key control and key rotation audit
  • S3 bucket policy restricts who can read log files
  • Do not grant S3 delete permissions to application roles — use S3 Object Lock (WORM) for immutability
  • CloudTrail itself cannot be disabled by non-root users if protected with an SCP
⚠ WATCH OUT An attacker with sufficient IAM permissions can delete a trail or disable logging. Protect CloudTrail with SCPs preventing trail deletion and CloudWatch alarm on StopLogging API calls.
OOrganization Trails & Multi-Account
  • Created in the AWS Organizations management account
  • Automatically applies to all current and future member accounts in the org
  • Member accounts can see the trail but cannot modify or delete it
  • All logs delivered to a central S3 bucket in the management or designated account
  • Prefix per account: s3://bucket/AWSLogs/<OrgID>/<AccountID>/
  • Central security team can audit all accounts from one trail without per-account setup
⚡ EXAM TIP "Single audit trail across all AWS accounts in the organization" → Organization Trail from management account. Member accounts cannot disable it — this is a key security property.
