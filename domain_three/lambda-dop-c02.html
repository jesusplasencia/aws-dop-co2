DOP-C02
01 Aliases & Versions High Weight
Versions

Working code lives on $LATEST (mutable). Publishing creates an immutable snapshot with its own ARN — code + configuration are frozen.

You cannot edit a published version. Increment by publishing again.

Aliases

A named pointer (DEV / PROD / TEST) that maps to one or two versions. Aliases are mutable — you can reroute them at any time.

Aliases cannot point to other aliases (no chaining).

Canary Deployment Assign weighted traffic split on an alias (e.g. 95% → V1, 5% → V2). Enables gradual rollouts without DNS changes.
ARN format arn:aws:lambda:region:account:function:name:QUALIFIER — qualifier can be a version number or alias name.
★ Exam focus: Weighted alias routing IS the mechanism for blue/green and canary releases in Lambda. Know that only aliases support weights — versions do not. Also remember: aliases are mutable, versions are immutable.
02 Environment Variables Medium Weight
Format Key/value pairs — String type only. Injected at runtime, accessible without code changes.
Size limit 4 KB total across all keys + values combined. Exceeding this limit causes deployment failure.
Encryption at rest Encrypted by default with Lambda-managed key. Use AWS KMS CMK for custom key control (required for cross-account scenarios).
Encryption in transit Use KMS + Decrypt API in function code before use, combined with console encryption helpers.
Scope Env vars are version-specific — stored with each published version. Changing them on $LATEST doesn't affect published versions.
03 Concurrency & Throttling High Weight
Account Default

1,000 concurrent executions shared across all functions in the region. Can be increased via AWS Support.

Unreserved pool is shared by all functions without reserved concurrency.

Reserved Concurrency

Set a hard cap on a specific function. Guarantees it won't be starved and won't exceed the limit.

Setting to 0 effectively disables the function (all invocations throttled).

Throttle — Sync Returns HTTP 429 TooManyRequestsException to the caller immediately. Caller must implement retry logic.
Throttle — Async Lambda retries automatically (up to 6 hours with exponential backoff). After retries exhausted → message goes to Dead Letter Queue (DLQ) or Event Bridge on failure destination.
Provisioned Concurrency Pre-warms execution environments before invocations arrive. Eliminates cold starts. Application Auto Scaling can adjust it on schedule or by utilization metric.
★ Exam focus: Know the difference between Reserved Concurrency (cap/guarantee) and Provisioned Concurrency (pre-warm/no cold start). DOP-C02 tests throttle behavior — sync vs async responses are a common distractor question.

05 File Systems Mounting (EFS) Lower Weight
Prerequisite Lambda must run inside a VPC that has network access to the EFS mount targets.
Access mechanism Configure an EFS Access Point. Lambda mounts it to a local directory path during init time — before handler runs.
Connection limit risk Each function instance holds an open NFS connection. High concurrency = connection burst to EFS. Monitor and set reserved concurrency to cap this.
Cross-account EFS Requires VPC Peering between accounts + EFS File System Policy granting the Lambda execution role permissions in the remote account.

⚡ Quick Reference — Exam Traps
Alias ≠ Version Only aliases can have weighted routing. You cannot assign traffic weights directly to versions.
No alias → alias Aliases cannot point to other aliases — they point only to versions or $LATEST.
Sync throttle = 429 Synchronous invocations return 429 immediately. The caller is responsible for retry — Lambda does NOT auto-retry sync calls.
Async throttle = DLQ Lambda retries async throttled invocations internally. Only after all retries fail does it send to DLQ / failure destination.
Env vars 4 KB limit Combined across ALL keys and values. Store large config in SSM Parameter Store or Secrets Manager instead.
Reserved vs Provisioned Reserved = cap on max concurrency. Provisioned = pre-warmed instances (no cold start, billed even when idle).
EFS requires VPC Lambda must be VPC-attached to use EFS. VPC Lambda also loses default internet access — needs NAT Gateway or VPC endpoints.
