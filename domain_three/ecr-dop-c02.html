Elastic Container Registry

Fully managed Docker image registry — the storage layer of every AWS container CI/CD pipeline.

01

Core Concepts

◈ MEDIUM

Repository Types

  • Private — IAM-controlled, per-account/region
  • Public — gallery.ecr.aws, read by anyone
  • Storage backed by Amazon S3
  • Images are immutable once pushed (optional setting)
  • Supports multi-arch image manifests (OCI)

Access Control

  • All access governed by IAM
  • Resource-based policy — grant cross-account or cross-region pull
  • ECS/Fargate pulls via Task Execution Role
  • EC2 pulls via Instance Profile
  • CodeBuild pulls/pushes via its Service Role
Exam Tip
If an ECS task can't pull an image → check the Task Execution Role (Fargate) or EC2 Instance Profile for ecr:GetAuthorizationToken and ecr:BatchGetImage permissions.
02

Image Security & Compliance

◈◈ HIGH

Image Scanning

  • Basic Scanning — on push, uses CVE database (OS packages)
  • Enhanced Scanning — continuous, powered by Amazon Inspector; covers OS + application dependencies
  • Findings sent to EventBridge and Security Hub
  • Automate: trigger Lambda or SNS on CRITICAL finding

Encryption

  • Images encrypted at rest by default (AES-256)
  • Optional KMS encryption for customer-managed keys
  • In-transit always via HTTPS/TLS

Image Tag Immutability

  • Enable per repository
  • Prevents overwriting an existing tag (e.g. :latest)
  • Enforces traceability — every tag maps to exactly one digest
  • Best practice for production pipelines
Exam Tip
Enhanced Scanning + EventBridge = automated vulnerability response pipeline. Tag Immutability prevents the ":latest overwrite" anti-pattern. Both are common DOP-C02 security architecture questions.
03

Lifecycle Policies

◈ MEDIUM
Purpose: Automatically clean up old images to control storage costs. Rules are evaluated in priority order (lowest number first).
Rule Type Filter By Example
sinceImagePushed Age (days) Expire untagged images older than 14 days
imageCountMoreThan Count Keep only the 5 most recent tagged images matching prod-*
Untagged only Tag status Keep 1 untagged image, expire all others
Exam Tip
Lifecycle policies only expire images — they don't delete layers from S3 until no image references them. Use dry-run (test rules) before applying. Rules apply per repository, not account-wide.

CI/CD Pipeline Integration

◈◈ HIGH
Standard pattern: Code commit → CodeBuild builds & pushes image to ECR → ECS/Fargate pulls and deploys.
CodeCommit / GitHubSource trigger
→
CodeBuilddocker build + push
→
ECRimage stored

ECS / Fargatepull & deploy

CodeBuild Permissions Required

  • ecr:GetAuthorizationToken
  • ecr:BatchCheckLayerAvailability
  • ecr:PutImage
  • ecr:InitiateLayerUpload
  • ecr:UploadLayerPart
  • ecr:CompleteLayerUpload
  • All on CodeBuild Service Role

Tagging Strategy (Best Practice)

  • Tag with Git commit SHA for traceability
  • Tag with semantic version (e.g. v1.2.3)
  • Use :latest only for dev/testing
  • Never use :latest in production Task Definitions
  • Enable Tag Immutability in prod repos
Exam Tip
CodePipeline triggers CodeBuild on source change → CodeBuild authenticates to ECR using aws ecr get-login-password piped to docker login. The authorization token is valid for 12 hours.
05

Cross-Account Access & Replication

◈ MEDIUM

Cross-Account Pull

  • Attach a Resource-based Policy to the ECR repository
  • Grant ecr:BatchGetImage + ecr:GetDownloadUrlForLayer to target account principal
  • No need to copy images — just share access
  • Common in multi-account CI/CD (build account → deploy account)

Replication

  • Cross-region replication — automatically mirror images to other regions
  • Cross-account replication — push to another AWS account's ECR
  • Configured at registry level (not per repo)
  • Used for DR, latency reduction, multi-region deployments
Exam Tip
Multi-account pipeline pattern: central build account pushes to ECR → resource-based policy allows dev/staging/prod accounts to pull. Replication is for geo-distribution or DR — not the primary access pattern.
06

Monitoring & Events

◈ LOW–MED

EventBridge Events

  • ECR Image Action — fires on push, delete, replicate
  • ECR Scan Action — fires when scan completes with findings
  • Route to SNS, Lambda, SQS for automation

CloudTrail

  • All ECR API calls logged automatically
  • Audit who pushed/pulled/deleted images
  • Key for compliance & forensics

CloudWatch Metrics

  • RepositoryPullCount — track pull volume
  • Storage metrics via S3 (underlying)
  • Scan finding severity counts
