01

The Two Capabilities

CodeGuru Reviewer

Static Code Intelligence

Analyzes your source code using ML on every pull request. Detects critical issues before they reach production. Not a linter — it understands patterns, security vulnerabilities, and AWS best practices from millions of real code reviews.

Triggers on PR · No agent · Repo integration
CodeGuru Profiler

Runtime Performance Intelligence

Continuously monitors your running application. Identifies the exact lines of code consuming the most CPU and memory in production — not just that your app is slow, but precisely which method is responsible.

Always-on · Agent required · Flame graphs

CodeGuru Reviewer — What It Analyzes

🔐
Security Vulnerabilities
Detects hardcoded credentials, secrets, and API keys left in source code. Catches passwords assigned to variables, connection strings with embedded credentials, and improperly handled cryptographic operations.
Secrets Detector — 40+ secret types

AWS Best Practices
Identifies incorrect usage of AWS SDKs — not reusing SDK clients across Lambda invocations, missing pagination handling, or improper exception handling for AWS API calls.

Code Quality Issues
Detects resource leaks (open file handles, unclosed connections), incorrect input validation, race conditions in concurrent code, and dead code paths.

Scans beyond application code — .env files, YAML configs, .properties files, and Infrastructure-as-Code templates (CloudFormation, Terraform).

CodeGuru Profiler — Runtime Intelligence

🔥
CPU Utilization Hotspots
Identifies which methods consume the most CPU time across all invocations. Reveals inefficient algorithms, unnecessary serialization/deserialization, or redundant computations that compound under load.
💾
Heap & Memory Analysis
Shows memory allocation patterns — which objects are allocated most frequently, helping identify memory leaks or excessive garbage collection pressure that causes latency spikes.
📊
Anomaly Detection
Automatically compares current profiles against historical baselines. If a new deployment increases CPU usage 30% for a specific method, CodeGuru Profiler flags it as an anomaly — no threshold configuration required.
💰
Cost Optimization Recommendations
The most exam-relevant Profiler feature. Identifying CPU-expensive code directly translates to cost savings — less compute time means lower EC2 instance hours or Lambda invocation costs.
Cost angle — specifically tested on DOP-C02

⚠️
Hard Constraint

CodeGuru Profiler supports Java and Python ONLY. This is a hard architectural constraint that appears in exam questions. If the scenario describes a Node.js, .NET, or Go application needing runtime profiling — CodeGuru Profiler is not the answer.

🎯
Reviewer vs Macie

Both deal with sensitive data — but Macie scans S3 buckets and objects. CodeGuru Reviewer scans source code repositories. Location is the differentiator.

🎯
Profiler vs X-Ray

X-Ray traces requests across distributed services — call chain and latency per service. CodeGuru Profiler shows which lines of code consume the most CPU inside one application. Complementary, not interchangeable.

🎯
Profiler vs CloudWatch

CloudWatch gives aggregate instance-level metrics (CPU % overall). CodeGuru Profiler gives code-level attribution (method X = 40% of CPU). When root cause at code level is needed — CloudWatch is wrong.

⚠️
Reviewer vs Linters

CodeGuru Reviewer is NOT a linter. Linters check syntax and style rules. Reviewer uses ML to understand patterns and context — it catches vulnerabilities and AWS-specific issues linters cannot detect.
