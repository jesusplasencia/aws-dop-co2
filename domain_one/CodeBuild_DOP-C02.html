01

The 5 Things to Know Cold

1
post_build always runs
Even when the build phase fails. Gate destructive actions with CODEBUILD_BUILD_SUCCEEDING.
2
VPC = NAT not IGW
Private subnet needs NAT Gateway for internet access. IGW doesn't reach private subnets.
3
Secrets via SSM/SM
Never hardcode. parameter-store and secrets-manager blocks in the env section.
4
Privileged Mode
Required for docker build commands. Explicit flag on the build project environment settings.
5
EventBridge trigger
CodeCommit push → EventBridge → CodeBuild. Add Lambda in middle for custom logic.
02

buildspec.yml — Complete Reference

Must live at the root of your source directory. For monorepos, override the path in the CodeBuild project configuration.

buildspec.ymlYAML
version: 0.2

env:
  shell: bash
  variables:
    ENVIRONMENT: "staging"
  parameter-store:                 # pulled from SSM Parameter Store
    DB_HOST: /myapp/prod/db_host
    LOGIN_PWD: /CodeBuild/dockerLoginPassword
  secrets-manager:                 # SecretName:JSONKey format
    API_KEY: MySecret:api_key
  exported-variables:              # share with downstream pipeline stages
    - ARTIFACT_VERSION
    - BUILD_TIMESTAMP

phases:
  install:
    runtime-versions:
      java: corretto17               # pin runtime — reproducible builds
      python: 3.11
    commands:
      - apt-get update -y
      - apt-get install -y maven
    finally:                         # always runs even on failure
      - echo "Install done"

  pre_build:
    commands:
      - docker login -u User -p $LOGIN_PWD
      - mvn validate

  build:
    commands:
      - mvn package -DskipTests
      - export ARTIFACT_VERSION=$(mvn help:evaluate \
          -Dexpression=project.version -q -DforceStdout)
    finally:
      - echo "Build done"

  post_build:                       # ALWAYS RUNS — even when build fails
    commands:
      - |
        if [ "$CODEBUILD_BUILD_SUCCEEDING" = "1" ]; then
          docker push $ECR_REGISTRY/myapp:$ARTIFACT_VERSION
        fi
    finally:
      - echo "Cleanup"

reports:
  UnitTestResults:
    files: ["jest-results.xml"]
    file-format: JUNITXML
  CoverageReport:
    files: ["coverage/lcov.info"]
    file-format: CLOVERXML
    coverage-enabled: yes

artifacts:
  files: ["target/*.jar", "appspec.yml", "scripts/**/*"]
  name: MyApp-$(date +%Y-%m-%d)      # dynamic naming

cache:
  paths:
    - "/root/.m2/**/*"              # Maven cache
    - "/root/.npm/**/*"
⚠️
Critical Trap

post_build runs even when the build phase fails. If post_build pushes a Docker image without checking build status, a broken image goes to production. Always gate with: if [ "$CODEBUILD_BUILD_SUCCEEDING" = "1" ]; then ... fi

04

VPC Access — The Critical Trap

By default CodeBuild runs outside your VPC — it has internet but cannot reach private resources (RDS, ElastiCache). When you move it inside the VPC, it loses internet access.

# Default (no VPC config)
✓ Public internet (npm, Maven, DockerHub)
✗ Private resources (RDS, ElastiCache, internal APIs)
 
# With VPC config — private subnet
✓ Private resources (RDS, ElastiCache, internal APIs)
✗ Public internet — LOST by default
 
# Correct solution: VPC + NAT Gateway in public subnet
CodeBuild (private subnet) → NAT Gateway (public subnet) → Internet
CodeBuild (private subnet) → RDS / ElastiCache (direct, same VPC)
⚠️
IGW Does Not Work

An Internet Gateway only serves resources in public subnets. CodeBuild runs in a private subnet — private subnets have no route to the IGW. The answer is always NAT Gateway. Not IGW. Not VPC endpoint alone.

ScenarioSolution
Private resources onlyVPC config only — no NAT needed
Public internet onlyDefault config — no VPC needed
Both private + public internetVPC + NAT Gateway in public subnet
AWS services (S3, ECR) without internetVPC + VPC Endpoints for those services

Build Caching & Local Debugging

Cache Strategies

TypeBest For
S3 CachePersist cache across all build environments — shared across projects
LOCAL_SOURCE_CACHEReduce source checkout time on repeat builds
LOCAL_DOCKER_LAYER_CACHESlow Docker builds — eliminates base image re-downloads
LOCAL_CUSTOM_CACHEMaven .m2, npm node_modules, Gradle caches
🎯
Exam Pattern

"Docker builds re-downloading base images every build." Answer: LOCAL_DOCKER_LAYER_CACHE — not a larger instance, not S3 cache.

"Prevent deploy when coverage drops below 80%"→CodeBuild test reports with coverage threshold
"Broken Docker image pushed despite failed build"→post_build ran — missing CODEBUILD_BUILD_SUCCEEDING check
"Debug failing build without AWS build costs"→CodeBuild local agent — Docker image
"Run unit + integration + E2E tests simultaneously"→CodeBuild batch builds with build-matrix
